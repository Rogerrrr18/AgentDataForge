# 任务：2048 滑块合并

用**单个 `index.html` 文件**（内联 JS，Canvas 或 DOM 均可）实现完整的 2048 游戏。

## 需求

1. **4×4 棋盘**，初始随机两格各放一个 `2` 或 `4`（`2` 概率约 90%，`4` 约 10%）。
2. **方向键**（← → ↑ ↓）滑动所有方块；同值相邻方块**合并翻倍**，方向上压缩。
3. 每次有效移动后，在随机空格**生成一个新块**（2 或 4）。
4. 出现 **2048** 即**胜利**；棋盘满且无可合并时**失败**；两种情况都能重新开始。
5. **滑动/合并有过渡动画**（方块位移、合并放大等），不是瞬移。
6. 显示当前**分数**（合并时累加合并后的值）；单文件、零外部依赖。

## 测试钩子（为自动评分，必须实现）

请在全局暴露游戏状态，供 rubric 读取/注入：

```js
window.__GAME = {
  board: <4x4 二维数组，0 表示空，否则为方块数值>,
  score: <number>,
  over: <boolean>,
  won:  <boolean>,
  setBoard: <(grid) => void>,   // 注入指定棋盘，用于确定性测试
};
```

`board` 需随每次移动实时更新。**未实现 `setBoard` 会被判 PARTIAL（降置信）——它是精确校验合并逻辑的关键。**

## 交付

把成品写到 `deliverables/index.html`（双击即可在浏览器打开运行）。

## 评分会检查（rubric）

- 注入已知棋盘 `[[2,2,0,0], ...]` 后按 ←，左上角变为 `4`（**精确合并逻辑**，可判对错）
- 有效移动后非空格数 +1（新块生成）
- 方向键响应、滑动动画
- 2048 胜利 / 满盘失败判定
- 视觉裁判打分：滑动动画流畅度
