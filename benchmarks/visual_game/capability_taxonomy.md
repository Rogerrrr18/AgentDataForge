# 能力分类法（Capability Taxonomy）

视觉/游戏代码生成专用的能力契约，5 族 23 项。每个任务的 `task.yaml` 里 `capabilities` 字段引用这些 id。

## 代码工程 CE

| id | 能力 | 说明 |
|---|---|---|
| CE-1 | HTML/CSS 布局与样式 | 纯结构/样式实现，无 JS 也能成型 |
| CE-2 | Canvas 2D 渲染 | 正确使用 Canvas 2D API 绘制与更新 |
| CE-3 | WebGL / Three.js 3D 渲染 | 3D 场景、着色器、相机 |
| CE-4 | 游戏循环与状态机 | 稳定的 update/render 循环 + 状态管理 |
| CE-5 | 交互与输入处理 | 事件监听、点击/键盘/拖拽响应 |
| CE-6 | 性能与帧率优化 | 掉帧控制、批量更新 |

## 数学物理 MP

| id | 能力 | 说明 |
|---|---|---|
| MP-1 | 几何与三角 | 旋转、角度、碰撞几何 |
| MP-2 | 线性代数（向量/矩阵） | 3D 变换、向量运算 |
| MP-3 | 物理仿真 | 运动学、重力、碰撞响应 |
| MP-4 | 粒子系统 | 粒子数组、批量更新、生命周期 |
| MP-5 | 概率与随机 | 随机生成、概率分布 |

## 视觉设计 VD

| id | 能力 | 说明 |
|---|---|---|
| VD-1 | 色彩与配色 | 调色板协调、可区分 |
| VD-2 | 构图与版式 | 布局清晰、信息层次 |
| VD-3 | 动画时序与缓动 | 持续动效、节奏、缓动曲线 |
| VD-4 | 排版 | 字重、行高、字号层次 |
| VD-5 | 审美判断 | 整体观感、现代感 |

## 推理 RS

| id | 能力 | 说明 |
|---|---|---|
| RS-1 | 空间推理 | 2D/3D 空间关系 |
| RS-2 | 算法思维 | 寻路/搜索/合并等核心算法 |
| RS-3 | 规则执行 | 游戏规则正确实现 |
| RS-4 | 复杂指令遵循 | 按规格实现（含测试钩子等细节） |
| RS-5 | 调试与自我修正 | 多轮内修复自身错误（跨任务，看轨迹） |

## 整合创造 IC

| id | 能力 | 说明 |
|---|---|---|
| IC-1 | 多系统整合 | 渲染+物理+输入+UI 协同 |
| IC-2 | 创造力/原创性 | 视觉独特、有冲击力 |
| IC-3 | 用户体验设计 | 可玩性、反馈、易用 |
| IC-4 | 自适应/AI 行为 | NPC、难度调节 |
